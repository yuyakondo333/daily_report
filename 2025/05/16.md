## わかったこと
- シーダーでデータを流し込む際に使った `__in`について
    - SQLのWHERE FIELD IN ~ に相当
    - コードが簡潔になる
```
user_ids = [1, 3, 5]
# ID が 1, 3, 5 のユーザーを検索
users = CustomUser.objects.filter(id__in=user_ids)
```
- シーダーを実行する際は以下のことをする
    - 値を取得して変数に格納
    - もし変数が空だったらメッセージ＋新しく値を作成する
    - 一括作成の際に`bulk_create`を使用する
